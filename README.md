# Sprint 3: Desarrollo y Despliegue de Contratos Inteligentes

## 📌 Definición del Sprint

En este sprint se desarrollará un contrato inteligente utilizando Soroban sobre la red Stellar Testnet.

El contrato tendrá como objetivo permitir el registro de servicios dentro de la plataforma LOC-SERV, almacenando información básica como el nombre del servicio, descripción y responsable.

Se busca que el contrato sea completamente funcional, que pueda ser desplegado en la red de prueba (Testnet) y que permita su interacción desde una aplicación externa, como una interfaz web o scripts.

---

## ✅ Criterios de Aceptación

- El contrato inteligente compila sin errores.
- La lógica del contrato permite registrar y consultar servicios.
- Se aplican buenas prácticas de desarrollo en blockchain.
- Se implementan pruebas unitarias básicas.
- El contrato es desplegado correctamente en la red Stellar Testnet.
- Se obtiene el Contract ID después del despliegue.
- El contrato puede ser utilizado desde una aplicación externa (frontend o script).
- Se documenta el uso e implementación del contrato en el repositorio.

---

## ⚙️ Configuración del Entorno

Para el desarrollo del contrato inteligente se configuró el siguiente entorno:

- Instalación del lenguaje Rust para el desarrollo del contrato.
- Instalación de Soroban CLI para la compilación y despliegue.
- Configuración de acceso a la red Stellar Testnet.
- Uso de la billetera Freighter para la firma de transacciones.
- Verificación de conexión con la red de prueba.

El entorno quedó listo para iniciar el desarrollo, pruebas y despliegue del contrato inteligente.
